# Judul Materi Belajar Analisis Kebutuhan dan Perancangan Antarmuka: Dasar sampai Mahir

Daftar judul materi ini disusun berdasarkan referensi pada bagian **"Referensi CPMK1 – Analisis Kebutuhan dan Perancangan Antarmuka (UI/UX)"** di [sumber_materi.md](sumber_materi.md), serta kurikulum standar dari MDN Accessibility, web.dev Learn Accessibility, FreeCodeCamp UI/UX, dan W3C WAI. Urutan dari dasar sampai mahir mengikuti alur: pengenalan kebutuhan pengguna → kebutuhan fungsional dan non-fungsional → wireframe, mockup, dan user flow → aksesibilitas dan UX → praktik lanjut dan alat.

---

## Tingkat Dasar

1. Pengenalan analisis kebutuhan dalam pengembangan solusi web
2. Kebutuhan fungsional dan non-fungsional: definisi dan perbedaan
3. Pengenalan UI (User Interface) dan UX (User Experience) serta perannya dalam pengembangan web
4. Mengenal pengguna (user) dan konteks penggunaan
5. Pengenalan wireframe dan mockup
6. Perbedaan low-fidelity dan high-fidelity dalam desain
7. Pengenalan user flow dan alur pengguna
8. Pengenalan aksesibilitas (a11y) dan pengalaman pengguna (UX) untuk web

---

## Tingkat Menengah

1. Penggalian kebutuhan (elicitation) dan wawancara pengguna
2. Mendokumentasikan kebutuhan fungsional dan non-fungsional
3. Analisis tugas (task analysis) dan pemetaan user flow
4. Membuat wireframe (low-fidelity) untuk halaman web
5. Membuat mockup (high-fidelity) dari wireframe
6. Prinsip desain visual dasar: layout, hierarki, dan konsistensi
7. Prinsip pengalaman pengguna: usability dan learnability
8. Aksesibilitas web: prinsip WCAG dan HTML semantik
9. Prototyping dan iterasi desain
10. Alat desain: pengenalan Figma (atau alternatif) untuk wireframe dan mockup

---

## Tingkat Lanjut / Mahir

1. Prioritisasi dan validasi kebutuhan (MoSCoW, acceptance criteria)
2. User flow lanjutan dan skenario pengguna
3. Design system dan konsistensi antarmuka
4. Aksesibilitas lanjutan: audit WCAG, ARIA, dan pengujian a11y
5. UX research dan usability testing
6. Handoff desain ke pengembang: spesifikasi dan komponen
7. Praktik terbaik desain dan aksesibilitas untuk produk web
8. Referensi standar: W3C WAI, web.dev Accessibility, MDN Accessibility

---

## Referensi

Materi detail untuk setiap judul di atas dapat dipelajari dari referensi berikut (lihat bagian **"Referensi CPMK1 – Analisis Kebutuhan dan Perancangan Antarmuka (UI/UX)"** di [sumber_materi.md](sumber_materi.md)):

- **AltexSoft – Functional and Non-functional Requirements** — Spesifikasi dan tipe kebutuhan fungsional vs non-fungsional untuk solusi perangkat lunak
- **Baeldung – Functional vs Non-functional Requirements** — Perbedaan kebutuhan fungsional dan non-fungsional beserta contoh
- **FreeCodeCamp – UI/UX Design Tutorial (Figma)** — Tutorial wireframe, prototype, dan desain antarmuka dengan Figma dari dasar
- **Lucidchart – How to Make a Wireframe** — Panduan langkah membuat wireframe
- **Interaction Design Foundation – Create Wireframes** — Pembuatan wireframe dan prinsipnya dalam desain UX
- **Figma** — Alat desain UI/UX untuk wireframe dan mockup (berbasis web)
- **MDN – Accessibility (Learn web development)** — Modul belajar aksesibilitas web: HTML semantik, CSS, JavaScript, dan ARIA
- **web.dev – Learn Accessibility** — Kursus aksesibilitas daring oleh Google untuk pengembang web
- **W3C WAI – Web Accessibility Initiative** — Standar dan panduan aksesibilitas W3C (WCAG, WAI-ARIA)
- **web.dev – Design and user experience** — Prinsip desain dan pengalaman pengguna (UX) untuk aksesibilitas web
