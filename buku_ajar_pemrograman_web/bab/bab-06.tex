\begin{aktivitas}
  \item \textbf{Praktik:} Buat tabel data lengkap dengan \code{<thead>}, \code{<tbody>}, \code{<tfoot>}, \code{scope}, dan \code{<caption>}; pastikan aksesibel menggunakan pembaca layar atau navigasi keyboard.
  \item \textbf{Praktik:} Buat form pendaftaran dengan berbagai tipe input (text, email, date, radio, checkbox, select, textarea), validasi dasar (\code{required}, \code{pattern}, \code{minlength}), serta \code{<fieldset>} dan \code{<legend>} untuk pengelompokan.
  \item \textbf{Proyek mini:} Rancang satu halaman lengkap dengan elemen semantik (\code{header}, \code{nav}, \code{main}, \code{article}, \code{aside}, \code{footer}); sisipkan media audio/video dengan subtitle track, SVG inline, dan iframe yang aman (sandbox).
  \item \textbf{Latihan:} Jalankan W3C Markup Validation Service pada halaman Anda, perbaiki semua error, dan dokumentasikan peringatan yang tersisa beserta alasannya.
  \item \textbf{Studi kasus:} Lakukan audit aksesibilitas menggunakan Lighthouse atau aXe DevTools; identifikasi dan perbaiki masalah yang ditemukan, lalu bandingkan skor sebelum dan sesudah perbaikan.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan antara \code{scope="col"} dan mekanisme \code{id}+\code{headers} pada tabel. Kapan sebaiknya menggunakan masing-masing pendekatan?
  \item Buat tabel perbandingan antara validasi client-side (HTML5) dan server-side. Mengapa keduanya diperlukan?
  \item Tambahkan \code{<fieldset>}/\code{<legend>} pada sebuah form dan uji hasilnya dengan navigasi keyboard saja. Catat perbedaan pengalaman sebelum dan sesudah penambahan.
  \item Jelaskan mengapa atribut \code{alt} pada gambar, \code{<label>} pada form, dan \code{<caption>} pada tabel penting untuk aksesibilitas. Berikan contoh kasus di mana ketidakhadiran masing-masing elemen ini menyulitkan pengguna.
  \item Apa yang dimaksud dengan prinsip POUR dalam WCAG? Berikan satu contoh penerapan untuk setiap prinsip.
  \item \textbf{Refleksi:} Dari materi bab ini, manakah indikator Sub-CPMK 2.1 dan Sub-CPMK 1.3 yang sudah Anda kuasai? Area mana yang masih perlu Anda tingkatkan untuk mencapai kompetensi penuh?
\end{latihan}

\begin{asesmen}
Asesmen Bab 6 merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 2.1 dan Sub-CPMK 1.3. Terkait RPS: Praktikum 2 (Pertemuan 5). Mahasiswa diminta membuat satu halaman web lengkap yang mengandung:
\begin{enumerate}
  \item Tabel data dengan pengelompokan (\code{thead}/\code{tbody}/\code{tfoot}), \code{scope}, dan \code{caption}. Mengukur Sub-CPMK 2.1.
  \item Form pendaftaran dengan minimal 5 tipe input berbeda, \code{fieldset}/\code{legend}, validasi HTML5, dan atribut aksesibilitas (label, aria-describedby). Mengukur Sub-CPMK 2.1.
  \item Elemen semantik yang tepat untuk struktur halaman. Mengukur Sub-CPMK 2.1.
  \item Media (audio atau video) dengan subtitle track dan teks alternatif. Mengukur Sub-CPMK 2.1.
  \item Hasil validasi W3C tanpa error dan skor aksesibilitas Lighthouse minimal 90. Mengukur Sub-CPMK 1.3.
\end{enumerate}

Komponen penilaian: kelengkapan fitur (30\%), ketepatan semantik dan standar (25\%), aksesibilitas (25\%), serta kualitas kode dan dokumentasi (20\%).

\textbf{Rubrik Penilaian (berbasis OBE):}
\begin{itemize}
  \item \textbf{A (85--100)}: Seluruh komponen lengkap, form dan tabel aksesibel, media terpasang benar, W3C valid, Lighthouse $\geq$90. Kompeten untuk Sub-CPMK 2.1 dan 1.3.
  \item \textbf{B (70--84)}: Sebagian besar komponen lengkap, beberapa detail aksesibilitas kurang.
  \item \textbf{C (55--69)}: Komponen dasar ada tetapi kurang mendalam.
  \item \textbf{D ($<$55)}: Komponen tidak lengkap atau tidak memenuhi standar minimum.
\end{itemize}
\end{asesmen}

\begin{rangkuman}
Bab ini membahas HTML form, media, validasi dasar, dan aksesibilitas form sebagai \textbf{Unit Materi Inti} berbasis Sub-CPMK 2.1 dan Sub-CPMK 1.3. Tabel HTML dilengkapi dengan pengelompokan (\code{thead}, \code{tbody}, \code{tfoot}), atribut \code{scope}/\code{headers} untuk aksesibilitas, dan \code{caption}. Form HTML didukung oleh beragam tipe input HTML5 dengan validasi bawaan browser, serta pengelompokan field menggunakan \code{fieldset}/\code{legend}. Media HTML5 (audio/video) dengan subtitle track dan SVG memperkaya konten. Embedding melalui iframe dengan atribut keamanan (sandbox) memungkinkan integrasi konten eksternal secara aman.

Aksesibilitas web dibangun di atas prinsip POUR (WCAG): konten harus perceivable, operable, understandable, dan robust. Atribut ARIA melengkapi semantik HTML saat diperlukan. Validasi dan debugging HTML menggunakan W3C Validator dan DevTools memastikan kode mengikuti standar.

\textbf{Poin Kunci:}
\begin{itemize}
  \item Tabel dan form membutuhkan struktur dan atribut yang mendukung aksesibilitas (scope, label, fieldset, ARIA).
  \item Media dan embedding perlu teks alternatif, subtitle, dan kontrol keamanan yang sesuai.
  \item Aksesibilitas (POUR/WCAG) memastikan web dapat digunakan oleh semua orang, termasuk pengguna dengan disabilitas.
  \item Validasi dan debugging HTML memastikan dokumen mengikuti standar dan bebas error.
  \item Best practices dan standar WHATWG/W3C menjadi panduan pembuatan kode HTML yang profesional dan interoperable.
\end{itemize}

\end{rangkuman}
