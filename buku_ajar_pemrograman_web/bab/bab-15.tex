\chapter{Integrasi API, Pengujian, Performa, Keamanan (XSS/CSRF), Deployment Dasar, Administrasi Web, dan Presentasi Hasil}

Bab ini membahas materi Pertemuan 15 sesuai RPS: integrasi API, pengujian, performa, keamanan (XSS/CSRF), deployment dasar, administrasi web, dan presentasi hasil. Materi meliputi integrasi API, pengujian aplikasi web (fungsional, performa, load testing), keamanan web (validasi input, SQL injection, XSS, CSRF, session management), deployment ke produksi, konfigurasi server/database, hardening, CI/CD, pemantauan, serta presentasi hasil proyek.

\begin{subcpmk}
  \item Sub-CPMK 3.3: Mengintegrasikan API dan menghubungkan front-end dengan back-end (request/response, data terstruktur).
  \item Sub-CPMK 4.1: Melakukan pengujian dan evaluasi performa aplikasi web.
  \item Sub-CPMK 4.2: Menerapkan prinsip keamanan web: validasi input, prepared statement, session, mitigasi XSS dan CSRF.
  \item Sub-CPMK 4.3: Melakukan deployment ke lingkungan produksi dan administrasi dasar (server/web, basis data).
\end{subcpmk}

\begin{aktivitas}
  \item \textbf{Praktik:} Buat endpoint sederhana yang mengembalikan JSON dan tampilkan di halaman web menggunakan Fetch API.
  \item \textbf{Praktik:} Lakukan pengujian fungsional pada form yang dibuat: skenario positif dan negatif.
  \item \textbf{Praktik:} Ukur performa halaman menggunakan Lighthouse (DevTools) dan catat skor serta rekomendasi.
  \item \textbf{Praktik:} Terapkan validasi input di server (PHP) pada form yang sudah dibuat.
  \item \textbf{Praktik:} Ganti query biasa menjadi prepared statement pada kode PHP yang mengakses database.
  \item \textbf{Praktik:} Implementasikan CSRF token pada form submit.
  \item \textbf{Praktik:} Terapkan \code{htmlspecialchars()} pada semua output yang berasal dari input pengguna.
  \item \textbf{Studi kasus:} Lakukan audit keamanan sederhana menggunakan checklist OWASP Top 10.
  \item \textbf{Praktik:} Bandingkan konfigurasi PHP di lingkungan lokal (XAMPP) dan produksi; catat perbedaannya.
  \item \textbf{Praktik:} Siapkan file \code{.env} untuk menyimpan konfigurasi database dan mode aplikasi.
  \item \textbf{Praktik:} Lakukan deployment sederhana ke server (atau simulasi lokal) dan dokumentasikan langkahnya.
  \item \textbf{Praktik:} Atur konfigurasi PHP produksi: nonaktifkan \code{display\_errors}, aktifkan \code{log\_errors}.
  \item \textbf{Praktik:} Buat skrip backup database menggunakan \code{mysqldump} dan uji restore-nya.
  \item \textbf{Latihan:} Jelaskan konsep reverse proxy dan HTTPS; identifikasi manfaatnya.
  \item \textbf{Latihan:} Buat checklist hardening server sederhana berdasarkan materi bab ini.
  \item \textbf{Diskusi:} Presentasikan proyek akhir dan lakukan refleksi kompetensi.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan pengujian fungsional, kompatibilitas, dan performa.
  \item Sebutkan tiga metrik performa web dan alat untuk mengukurnya.
  \item Jelaskan mengapa validasi di client saja tidak cukup untuk keamanan.
  \item Tulis contoh prepared statement dengan MySQLi untuk query INSERT.
  \item Jelaskan perbedaan XSS dan CSRF beserta cara pencegahannya.
  \item Sebutkan tiga atribut cookie untuk keamanan dan fungsi masing-masing.
  \item Jelaskan tiga perbedaan utama antara lingkungan development dan produksi.
  \item Buat checklist langkah deployment dari lokal ke server.
  \item Sebutkan lima pengaturan \code{php.ini} yang harus diubah untuk produksi.
  \item Jelaskan fungsi environment variable dan mengapa file \code{.env} tidak boleh masuk ke Git.
  \item Jelaskan konsep CI/CD dan sebutkan dua alat yang mendukungnya.
  \item Sebutkan tiga hal yang harus dipantau pada aplikasi di produksi.
  \item \textbf{Refleksi:} Dari materi bab ini, manakah indikator Sub-CPMK 3.3, 4.1, 4.2, dan 4.3 yang sudah Anda kuasai? Area mana yang masih perlu Anda tingkatkan sebelum UAS dan Project?
\end{latihan}

\begin{asesmen}
Asesmen Bab 15 (Proyek Pertemuan 15) merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 3.3, 4.1, 4.2, dan 4.3. Proyek berupa aplikasi web terintegrasi yang mencakup:
\begin{enumerate}
  \item \textbf{Integrasi API} (Sub-CPMK 3.3): Endpoint JSON, Fetch API, request/response terstruktur.
  \item \textbf{Laporan Pengujian} (Sub-CPMK 4.1): Pengujian fungsional dan evaluasi performa (Lighthouse/DevTools).
  \item \textbf{Implementasi Keamanan} (Sub-CPMK 4.2): Prepared statement, CSRF token, output encoding (\code{htmlspecialchars}).
  \item \textbf{Deployment} (Sub-CPMK 4.3): Konfigurasi produksi, environment variable, backup/restore, dokumentasi.
  \item \textbf{Presentasi Hasil}: Presentasi proyek dan refleksi kompetensi.
\end{enumerate}

\textbf{Rubrik Penilaian (berbasis OBE):}
\begin{itemize}
  \item \textbf{A (85--100)}: Seluruh komponen lengkap, API terintegrasi dengan baik, keamanan dan deployment memenuhi standar. Kompeten untuk Sub-CPMK 3.3, 4.1, 4.2, 4.3.
  \item \textbf{B (70--84)}: Sebagian besar komponen lengkap, beberapa detail kurang.
  \item \textbf{C (55--69)}: Komponen dasar ada tetapi kurang mendalam.
  \item \textbf{D ($<$55)}: Komponen tidak lengkap atau tidak memenuhi standar minimum.
\end{itemize}
\end{asesmen}

\begin{rangkuman}
Bab ini merupakan \textbf{Unit Materi Inti} berbasis Sub-CPMK 3.3, 4.1, 4.2, dan 4.3 yang membahas integrasi API, pengujian, keamanan web, deployment, dan administrasi web secara menyeluruh. Keamanan dan pengujian bukan langkah terakhir; keduanya harus terintegrasi di setiap tahap pengembangan. Deployment ke produksi membutuhkan konfigurasi yang berbeda dari lingkungan lokal.

\textbf{Poin Kunci:}
\begin{itemize}
  \item \textbf{Integrasi API} (Sub-CPMK 3.3): API memudahkan pertukaran data antara front-end dan back-end; Fetch API dan endpoint JSON menghubungkan kedua layer.
  \item \textbf{Pengujian} (Sub-CPMK 4.1): Fungsional, kompatibilitas, performa; metrik FCP, LCP; Lighthouse dan DevTools.
  \item \textbf{Keamanan} (Sub-CPMK 4.2): Validasi input di server, prepared statement, output encoding, session/cookie aman, CSRF token, OWASP Top 10.
  \item \textbf{Deployment dan Administrasi} (Sub-CPMK 4.3): Environment variable, konfigurasi PHP produksi, reverse proxy/HTTPS, hardening, backup/restore, CI/CD, pemantauan.
\end{itemize}

\end{rangkuman}
