\begin{asesmen}
Asesmen Bab 4 mengevaluasi penguasaan struktur HTML, ketepatan penggunaan elemen, aksesibilitas, dan praktik terbaik. Komponen penilaian meliputi:

\begin{itemize}[leftmargin=*]
  \item \textbf{Struktur Dokumen (30\%):} Keberadaan doctype, html, head lengkap dengan meta wajib (charset, viewport, title), dan body.
  \item \textbf{Elemen Semantik (25\%):} Penggunaan header, nav, main, article/section, aside, footer sesuai konteks.
  \item \textbf{Konten dan Format (20\%):} Hierarki heading yang benar, paragraf, daftar, dan elemen teks yang tepat.
  \item \textbf{Link dan Gambar (15\%):} Atribut href yang valid, keamanan link eksternal (noopener noreferrer), atribut alt dan dimensi pada gambar.
  \item \textbf{Validasi dan Best Practice (10\%):} Passing W3C Validator, tidak ada overfull/underfull hbox parah, kode rapi dengan indentasi konsisten.
\end{itemize}

Rubrik penilaian detail tersedia di Lampiran untuk penilaian proyek halaman profil lengkap.
\end{asesmen}

\begin{rangkuman}
Bab ini telah membahas fondasi pembuatan halaman web dengan HTML5, mulai dari pengenalan peran HTML di ekosistem web hingga implementasi elemen semantik untuk struktur halaman modern. Materi yang dipelajari meliputi:

\begin{itemize}[leftmargin=*]
  \item \textbf{Pengenalan HTML:} HTML sebagai bahasa markup deklaratif untuk struktur dan makna konten, evolusi dari HTML 1.0 hingga HTML5 Living Standard, dan perannya bersama CSS dan JavaScript dalam triad teknologi web.
  \item \textbf{Setup dan Alat:} Editor kode (VS Code, Notepad++), browser modern dengan DevTools, workflow edit-save-refresh, dan struktur folder proyek.
  \item \textbf{Sintaks Dasar:} Perbedaan tag, elemen, dan atribut; void elements; aturan nesting; dan atribut global (id, class, lang, data-*).
  \item \textbf{Struktur Dokumen:} Doctype HTML5, elemen root html, bagian head (metadata, charset, viewport, title, link), dan bagian body (konten terlihat).
  \item \textbf{Konten Teks:} Heading h1-h6 dengan hierarki aksesibel, paragraf p, elemen semantik (em, strong, mark, del, ins), dan perbandingan dengan elemen presentasional (i, b).
  \item \textbf{Daftar:} Unordered list (ul, li), ordered list (ol dengan type, start, reversed), description list (dl, dt, dd), dan nested lists.
  \item \textbf{Fitur Lanjutan:} Kutipan (blockquote, q), blok kode (pre, code), subscript dan superscript (sub, sup), serta HTML entities (\&lt;, \&gt;, \&amp;, \&nbsp;, \&copy;).
  \item \textbf{Link:} Elemen a dengan href, link internal/eksternal/anchor, keamanan link (noopener noreferrer), dan praktik teks link deskriptif.
  \item \textbf{Gambar:} Elemen img dengan src, alt (penting untuk aksesibilitas), width/height (layout stability), dan figure/figcaption untuk keterangan semantik.
  \item \textbf{Elemen Semantik:} Header, nav, main, article, section, aside, footer untuk struktur halaman yang bermakna, aksesibel, dan SEO-friendly.
\end{itemize}

\textbf{Poin Kunci:}
\begin{enumerate}[leftmargin=*]
  \item HTML mendefinisikan struktur dan makna konten; struktur standar terdiri dari \code{head} (metadata) dan \code{body} (konten terlihat).
  \item Gunakan elemen yang tepat dengan atribut yang lengkap; prioritas elemen semantik (em, strong) atas elemen presentasional (i, b).
  \item Aksesibilitas adalah tanggung jawab pengembang: alt text untuk gambar, hierarki heading yang logis, dan elemen semantik untuk screen readers.
  \item Keamanan web dimulai dari detail kecil seperti \code{rel="noopener noreferrer"} pada link eksternal.
  \item Elemen semantik HTML5 meningkatkan SEO, aksesibilitas, dan maintainability kode secara signifikan dibanding div generik.
\end{enumerate}

Dengan menguasai konsep-konsep ini, Anda telah membangun fondasi yang kuat untuk melanjutkan pembelajaran ke topik HTML lanjutan (form, media, tabel) dan integrasi dengan CSS untuk styling.
\end{rangkuman}
