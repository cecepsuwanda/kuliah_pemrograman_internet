\chapter{Mockup, Prototyping, Aksesibilitas, dan Prinsip UX}

Bab ini melanjutkan materi Bab 3 dengan fokus pada mockup high-fidelity, prototyping, prinsip desain visual dan UX, aksesibilitas web (WCAG, HTML semantik, ARIA), design system, serta UX research dan usability testing. Materi ini mendukung pencapaian Sub-CPMK 1.2 dan 1.3 sesuai rencana pembelajaran pertemuan ketiga.

\begin{subcpmk}
  \item Sub-CPMK 1.2: Membuat wireframe dan mockup antarmuka web yang mendukung alur pengguna (user flow).
  \item Sub-CPMK 1.3: Menerapkan prinsip aksesibilitas (a11y) dan pengalaman pengguna (UX) dalam desain antarmuka.
\end{subcpmk}

\begin{aktivitas}
  \item \textbf{Praktik:} Kembangkan wireframe (dari Bab 3) menjadi mockup high-fidelity menggunakan Figma, termasuk palet warna, tipografi, dan spacing.
  \item \textbf{Praktik:} Buat prototipe interaktif dari mockup dengan minimal tiga halaman terhubung (navigasi klik).
  \item \textbf{Studi kasus:} Lakukan evaluasi heuristik (10 heuristik Nielsen) terhadap satu situs web pilihan dan dokumentasikan temuannya.
  \item \textbf{Praktik:} Audit aksesibilitas halaman web menggunakan Lighthouse dan perbaiki minimal tiga temuan (HTML semantik atau ARIA).
  \item \textbf{Praktik:} Lakukan usability testing sederhana dengan 3 rekan: siapkan tugas, amati, dan buat laporan temuan.
  \item \textbf{Latihan:} Buat dokumen design system sederhana: design tokens (warna, font, spacing), tiga komponen (tombol, input, kartu), dan panduan penggunaan.
  \item \textbf{Diskusi:} Presentasikan mockup dan prototipe di depan kelompok; terima umpan balik dan lakukan iterasi.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan wireframe dan mockup serta kapan masing-masing dibuat dalam alur desain.
  \item Identifikasi tiga pelanggaran heuristik Nielsen pada halaman web yang sering Anda kunjungi.
  \item Tuliskan kode HTML semantik untuk halaman kontak yang aksesibel (gunakan \code{<form>}, \code{<label>}, dan \code{aria-describedby}).
  \item Jelaskan empat prinsip POUR dalam WCAG dan berikan satu contoh implementasi untuk masing-masing.
  \item Apa itu design token? Tuliskan lima design token (variabel CSS) untuk sebuah design system sederhana.
  \item Rancang skenario usability testing sederhana: tentukan tujuan, tiga tugas untuk partisipan, dan metrik yang diukur.
  \item \textbf{Refleksi:} Dari materi bab ini, manakah indikator Sub-CPMK 1.2 dan 1.3 yang sudah Anda kuasai? Area mana yang masih perlu Anda tingkatkan?
\end{latihan}

\begin{asesmen}
Asesmen Bab 4 (Tugas 2) merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 1.2 dan Sub-CPMK 1.3. Tugas berupa portofolio yang mencakup:
\begin{enumerate}
  \item \textbf{Mockup High-Fidelity} (25\%): Satu halaman utama dari wireframe Bab 3 dikembangkan menjadi mockup di Figma dengan palet warna, tipografi, dan spacing yang konsisten. Mengukur Sub-CPMK 1.2.
  \item \textbf{Prototipe Interaktif} (15\%): Prototipe klik minimal tiga halaman dengan transisi yang jelas. Mengukur Sub-CPMK 1.2.
  \item \textbf{Laporan Aksesibilitas} (25\%): Audit WCAG menggunakan Lighthouse, identifikasi masalah, dan implementasi perbaikan (kode HTML semantik + ARIA). Mengukur Sub-CPMK 1.3.
  \item \textbf{Evaluasi Heuristik} (15\%): Laporan evaluasi 10 heuristik Nielsen terhadap satu situs dengan severity rating. Mengukur Sub-CPMK 1.2 dan 1.3.
  \item \textbf{Laporan Usability Testing} (20\%): Protokol testing, temuan dari minimal 3 partisipan, dan rekomendasi perbaikan. Mengukur Sub-CPMK 1.3.
\end{enumerate}

\textbf{Rubrik Penilaian (berbasis OBE):}
\begin{itemize}
  \item \textbf{A (85--100)}: Seluruh komponen lengkap, mockup dan prototipe profesional, aksesibilitas terimplementasi dengan baik. Kompeten untuk Sub-CPMK 1.2 dan 1.3.
  \item \textbf{B (70--84)}: Sebagian besar komponen lengkap, beberapa detail kurang.
  \item \textbf{C (55--69)}: Komponen dasar ada tetapi kurang mendalam.
  \item \textbf{D ($<$55)}: Komponen tidak lengkap atau tidak memenuhi standar minimum.
\end{itemize}
\end{asesmen}

\begin{rangkuman}
Bab ini membahas mockup, prototyping, prinsip UX, aksesibilitas, design system, dan validasi desain melalui UX research sebagai Unit Materi Inti berbasis Sub-CPMK 1.2 dan 1.3.

\textbf{Poin Kunci:}
\begin{itemize}
  \item \textbf{Mockup} (Sub-CPMK 1.2): high-fi memvalidasi tampilan akhir; dibuat setelah wireframe disetujui dengan alat seperti Figma.
  \item \textbf{Prinsip desain visual} (layout, hierarki, tipografi, warna) dan \textbf{10 heuristik Nielsen} menjadi panduan evaluasi antarmuka.
  \item \textbf{Aksesibilitas} (Sub-CPMK 1.3): WCAG/POUR, HTML semantik, ARIA memastikan web inklusif untuk semua pengguna.
  \item \textbf{Prototyping dan iterasi} menghasilkan desain yang terus membaik berdasarkan umpan balik pengguna.
  \item \textbf{Design system} menjaga konsistensi; \textbf{handoff} yang baik memperlancar transisi dari desain ke kode.
  \item \textbf{UX research dan usability testing} memvalidasi desain dengan pengguna nyata---cukup 5 partisipan untuk menemukan 85\% masalah.
\end{itemize}

\textbf{Refleksi:} Pastikan Anda dapat mencentang checklist di atas dan siap melanjutkan ke Bab 5 (HTML dasar, struktur dokumen, elemen semantik).
\end{rangkuman}
