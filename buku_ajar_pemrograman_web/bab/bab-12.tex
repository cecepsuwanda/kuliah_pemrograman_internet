\begin{aktivitas}
  \item \textbf{Praktik:} Instal XAMPP (atau server lokal lain) dan verifikasi instalasi PHP dengan membuat file \code{phpinfo()} lalu mengaksesnya melalui browser.
  \item \textbf{Praktik:} Tulis skrip PHP yang menggunakan variabel, tipe data, operator aritmetika, dan operator perbandingan untuk membuat kalkulator sederhana.
  \item \textbf{Praktik:} Praktikkan struktur kontrol: buat program yang menentukan grade (A/B/C/D/E) berdasarkan nilai numerik menggunakan \code{if-elseif-else} dan \code{switch}.
  \item \textbf{Latihan:} Buat program yang menggunakan perulangan \code{for} dan \code{foreach} untuk menampilkan data array dalam format tabel HTML.
  \item \textbf{Praktik:} Implementasikan form HTML lengkap (nama, email, pesan) dengan pemrosesan dan validasi server-side di PHP, termasuk sanitasi output dengan \code{htmlspecialchars()}.
  \item \textbf{Praktik:} Organisasikan proyek PHP sederhana menggunakan \code{include}/\code{require} untuk memisahkan header, footer, konfigurasi, dan fungsi ke file-file terpisah.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan mendasar antara metode GET dan POST, serta berikan contoh skenario kapan masing-masing metode sebaiknya digunakan.
  \item Apa perbedaan antara operator \code{==} dan \code{===} di PHP? Berikan contoh kasus di mana penggunaan \code{==} dapat menyebabkan bug.
  \item Buat fungsi PHP bernama \code{validasiEmail(\$email)} yang memeriksa apakah email tidak kosong dan memiliki format yang valid menggunakan \code{filter\_var()}.
  \item Jelaskan perbedaan antara \code{include}, \code{require}, \code{include\_once}, dan \code{require\_once}, serta kapan sebaiknya menggunakan masing-masing.
  \item Mengapa validasi server-side tetap diperlukan meskipun form sudah memiliki validasi HTML5 di sisi klien? Berikan contoh skenario keamanannya.
  \item \textbf{Refleksi:} Dari materi bab ini, indikator Sub-CPMK 3.1 mana yang sudah Anda kuasai? Bagian mana yang masih perlu Anda tingkatkan?
\end{latihan}

\begin{asesmen}
Asesmen Bab 12 (Praktikum 6) merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 3.1. Tugas berupa proyek membuat sistem form multi-halaman dengan PHP yang mencakup:
\begin{enumerate}
  \item \textbf{Form registrasi} (40\%): Halaman form registrasi (nama, email, password, konfirmasi password) dengan validasi server-side lengkap. Mengukur Sub-CPMK 3.1 (pemrosesan form, validasi).
  \item \textbf{Halaman hasil} (25\%): Halaman hasil yang menampilkan data yang berhasil divalidasi dengan sanitasi output (\code{htmlspecialchars()}). Mengukur Sub-CPMK 3.1.
  \item \textbf{Organisasi kode} (25\%): Penggunaan \code{include}/\code{require} untuk memisahkan header, footer, konfigurasi, dan fungsi validasi ke file terpisah. Mengukur Sub-CPMK 3.1.
  \item \textbf{Sticky form} (10\%): Implementasi teknik \textit{sticky form} agar data tidak hilang jika validasi gagal. Mengukur Sub-CPMK 3.1.
\end{enumerate}

\textbf{Rubrik Penilaian (berbasis OBE):}
\begin{itemize}
  \item \textbf{A (85--100)}: Seluruh komponen lengkap, validasi server-side benar, sanitasi output diterapkan, organisasi kode rapi. Kompeten untuk Sub-CPMK 3.1.
  \item \textbf{B (70--84)}: Sebagian besar komponen lengkap, beberapa detail kurang.
  \item \textbf{C (55--69)}: Komponen dasar ada tetapi kurang mendalam.
  \item \textbf{D ($<$55)}: Komponen tidak lengkap atau tidak memenuhi standar minimum.
\end{itemize}
\end{asesmen}

\begin{rangkuman}
Bab ini membahas PHP dasar, pemrosesan form, dan include/require sebagai \textbf{Unit Materi Inti} berbasis Sub-CPMK 3.1. Dimulai dari instalasi lingkungan (XAMPP), sintaks dasar (tag, komentar, aturan penulisan), variabel (penamaan, tipe dinamis, scope), tipe data (string, integer, float, boolean, null), dan operator (aritmetika, perbandingan, logika, penugasan).

Selanjutnya dibahas cara menampilkan output (echo/print), manipulasi string (literal, escape, heredoc, fungsi string), struktur kontrol percabangan (if/elseif/else, switch, ternary), dan perulangan (for, while, do-while, foreach). Fungsi memungkinkan penggunaan ulang kode dengan dukungan parameter default, type hinting, dan variadic arguments. Array (terindeks, asosiatif, bersarang) menjadi struktur data fundamental di PHP.

Bab ini ditutup dengan pembahasan superglobal (\code{\$\_GET}, \code{\$\_POST}, \code{\$\_SERVER}), pemrosesan form HTML dengan PHP, validasi server-side, dan pengorganisasian kode menggunakan \code{include}/\code{require}.

\textbf{Poin Kunci:}
\begin{itemize}
  \item PHP berjalan di server dan menghasilkan output HTML yang dikirim ke browser.
  \item Validasi server-side \textbf{wajib} dilakukan karena validasi klien dapat dilewati.
  \item Selalu sanitasi output dengan \code{htmlspecialchars()} untuk mencegah XSS.
  \item Gunakan \code{===} (identik) alih-alih \code{==} untuk perbandingan yang aman.
  \item Organisasikan kode dengan \code{include}/\code{require} mengikuti prinsip DRY.
\end{itemize}

\end{rangkuman}
