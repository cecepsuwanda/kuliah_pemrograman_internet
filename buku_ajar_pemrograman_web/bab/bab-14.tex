\begin{aktivitas}
  \item \textbf{Praktik:} Instalasi MySQL (via XAMPP); akses CLI dan phpMyAdmin. Buat database dan tabel dengan berbagai tipe data.
  \item \textbf{Praktik:} Praktik DDL: ALTER TABLE. Praktik DML: SELECT dengan WHERE, operator, ORDER BY, LIMIT; INSERT, UPDATE, DELETE.
  \item \textbf{Praktik:} Gunakan fungsi agregat (COUNT, SUM, AVG, MIN, MAX), GROUP BY, HAVING, DISTINCT, alias. Tangani nilai NULL.
  \item \textbf{Praktik:} Desain relasi tabel: primary key, foreign key, normalisasi (1NF--3NF). Praktik JOIN (INNER, LEFT, RIGHT) dan subquery.
  \item \textbf{Praktik:} Buat index dan analisis performa query dengan EXPLAIN. Praktik backup/restore dengan mysqldump.
  \item \textbf{Praktik:} Koneksi PHP--MySQL dengan MySQLi (OOP). Praktik fetch: \code{fetch\_assoc()}, \code{fetch\_all()}. Bangun aplikasi CRUD lengkap.
  \item \textbf{Praktik:} Gunakan prepared statement (MySQLi dan PDO) untuk mencegah SQL injection. Bandingkan MySQLi vs PDO.
  \item \textbf{Praktik:} Praktik MySQL lanjut: VIEW, stored procedure, trigger, transaksi (BEGIN/COMMIT/ROLLBACK).
  \item \textbf{Praktik:} Kelola user dan hak akses MySQL. Rancang skema database lengkap dengan diagram ER.
  \item \textbf{Praktik:} Terapkan keamanan web: validasi input, sanitasi output, mitigasi XSS dan CSRF dengan token.
  \item \textbf{Studi literatur:} Pelajari exception, namespace, Composer, dan struktur proyek PHP sesuai PSR.
  \item \textbf{Praktik:} Bangun REST API sederhana dengan PHP; hubungkan front-end (Fetch API) dengan back-end.
  \item \textbf{Praktik:} Tulis unit test dengan PHPUnit. Siapkan checklist deployment untuk produksi.
  \item \textbf{Studi kasus:} Integrasikan seluruh materi dalam Aplikasi Web CRUD Full-Stack (database, PHP, API, keamanan).
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan DDL dan DML; berikan contoh minimal 2 perintah masing-masing.
  \item Apa peran primary key dan foreign key dalam menjaga integritas data?
  \item Tulis query dengan JOIN untuk menggabungkan data dari dua tabel; jelaskan perbedaan INNER, LEFT, dan RIGHT JOIN.
  \item Apa itu normalisasi? Jelaskan 1NF, 2NF, dan 3NF dengan contoh.
  \item Jelaskan bagaimana prepared statement mencegah SQL injection dibandingkan penggabungan string.
  \item Bandingkan MySQLi dan PDO dari sisi portabilitas, fitur, dan gaya API.
  \item Buat endpoint REST API sederhana yang mengembalikan data produk dalam format JSON.
  \item Apa itu CSRF token dan mengapa diperlukan di setiap form? Jelaskan mekanismenya.
  \item \textbf{Refleksi:} Dari materi bab ini, indikator Sub-CPMK 3.2 mana yang sudah Anda kuasai? Area mana yang masih perlu Anda tingkatkan?
\end{latihan}

\begin{asesmen}
Asesmen Bab 14 (Praktikum 8) merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 3.2. Tugas berupa proyek: \textbf{Aplikasi Web CRUD Full-Stack} dengan komponen berikut; setiap komponen terhubung ke indikator pencapaian Sub-CPMK 3.2:

\begin{enumerate}
  \item \textbf{Desain database} (20\%): Database ternormalisasi (minimal 3 tabel dengan relasi FK) dan diagram ER. Mengukur kemampuan mengelola basis data (Sub-CPMK 3.2).
  \item \textbf{Implementasi CRUD} (25\%): Operasi Create, Read, Update, Delete dengan PHP dan MySQLi/PDO menggunakan prepared statement. Mengukur kemampuan mengimplementasikan CRUD (Sub-CPMK 3.2).
  \item \textbf{REST API} (15\%): Minimal 2 endpoint (GET dan POST) dengan respons JSON. Mengukur integrasi koneksi PHP--MySQL dan penyediaan data terstruktur (Sub-CPMK 3.2).
  \item \textbf{Integrasi front-end} (10\%): Penggunaan JavaScript Fetch API untuk menampilkan data dari back-end. Mengukur koneksi dari PHP (Sub-CPMK 3.2).
  \item \textbf{Keamanan} (15\%): Validasi input, sanitasi output, CSRF token, password hashing. Mendukung keandalan aplikasi CRUD.
  \item \textbf{Dokumentasi} (15\%): Laporan atau dokumentasi singkat. (Opsional) Pengujian unit PHPUnit; deployment checklist.
\end{enumerate}

\end{itemize}
\end{asesmen}

\begin{rangkuman}
Bab ini merupakan \textbf{Unit Materi Inti} berbasis Sub-CPMK 3.2 yang membahas MySQL dan operasi CRUD secara komprehensif dalam empat area utama. \textbf{MySQL Dasar} (section 1--14) mencakup pengenalan database dan SQL, instalasi MySQL, akses CLI dan GUI, DDL (CREATE DATABASE, CREATE TABLE, ALTER TABLE), DML (SELECT, INSERT, UPDATE, DELETE), operator, ORDER BY/LIMIT, fungsi agregat, GROUP BY/HAVING, DISTINCT/alias, serta penanganan NULL.

\textbf{Relasi dan Lanjut} (section 15--22) membahas relasi tabel dan konsep kunci (PK/FK), JOIN (INNER, LEFT, RIGHT), subquery, PRIMARY KEY vs UNIQUE, FOREIGN KEY dan referential integrity, normalisasi (1NF/2NF/3NF), index, serta backup dan restore dengan mysqldump.

\textbf{Integrasi PHP--MySQL} (section 23--27) mengajarkan koneksi MySQLi (OOP), metode fetch, CRUD aplikasi web dengan prepared statement, pencegahan SQL injection, serta PDO sebagai alternatif yang portable.

\textbf{MySQL Advanced dan PHP Lanjut} (section 28--46) meliputi VIEW, stored procedure, trigger, transaksi (ACID), optimasi EXPLAIN, keamanan user, perancangan database, keamanan web (validasi/sanitasi, XSS, CSRF), exception, namespace, Composer, struktur proyek, PHP CLI, REST API, integrasi front-back dengan Fetch API, pengujian PHPUnit, dan deployment.

\textbf{Poin Kunci:}
\begin{itemize}
  \item SQL DDL/DML, relasi, JOIN, normalisasi, dan index adalah fondasi pengelolaan basis data.
  \item Prepared statement (MySQLi/PDO) wajib untuk keamanan; jangan pernah gabungkan input ke query.
  \item REST API dengan JSON menjadi standar komunikasi front-end dan back-end modern.
  \item Keamanan, pengujian, dan deployment yang baik melengkapi siklus pengembangan full-stack.
\end{itemize}
