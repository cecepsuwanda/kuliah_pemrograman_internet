\begin{aktivitas}
  \item \textbf{Praktik:} Buat class PHP (\code{Mahasiswa}, \code{Produk}) dengan constructor, properti (typed), metode, dan demonstrasi visibility (public, protected, private).
  \item \textbf{Praktik:} Praktikkan inheritance: buat hierarki class (misal \code{Hewan} $\to$ \code{Kucing}, \code{Anjing}) dengan method overriding dan \code{parent::}. Buat juga class dengan static property dan factory method.
  \item \textbf{Praktik:} Bangun sistem login sederhana menggunakan session: \code{session\_start()}, \code{session\_regenerate\_id()}, \code{session\_destroy()}. Terapkan atribut keamanan cookie session (HttpOnly, Secure, SameSite).
  \item \textbf{Praktik:} Implementasikan form upload file dengan validasi lengkap (ukuran, ekstensi whitelist, MIME type) dan \code{move\_uploaded\_file()}.
  \item \textbf{Praktik:} Buat koneksi MySQLi (OOP) dan jalankan query CRUD menggunakan prepared statement (\code{prepare()}, \code{bind\_param()}, \code{execute()}).
  \item \textbf{Praktik:} Bangun form lanjutan dengan validasi server-side (\code{filter\_var()}, regex) dan sanitasi (\code{htmlspecialchars()}, \code{trim()}, \code{strip\_tags()}). Tambahkan CSRF token.
  \item \textbf{Studi kasus:} Integrasikan semua materi dalam Aplikasi Daftar Tugas (CRUD + Login + Session).
\end{aktivitas}

\begin{latihan}
  \item Jelaskan empat pilar OOP (encapsulation, inheritance, polymorphism, abstraction) dan berikan contoh masing-masing dalam konteks PHP.
  \item Apa perbedaan antara \code{public}, \code{protected}, dan \code{private}? Mengapa encapsulation penting?
  \item Jelaskan perbedaan session dan cookie, serta sebutkan kapan masing-masing sebaiknya digunakan.
  \item Apa itu \textit{prepared statement}? Mengapa lebih aman daripada menggabungkan string langsung ke query SQL?
  \item Terapkan \code{htmlspecialchars()} saat menampilkan input pengguna. Jelaskan risiko XSS dan bagaimana cara mitigasinya.
  \item Apa itu CSRF token dan bagaimana cara kerjanya untuk melindungi form dari serangan CSRF?
  \item \textbf{Refleksi:} Dari materi bab ini, indikator Sub-CPMK 3.1 dan 4.2 mana yang sudah Anda kuasai? Area mana yang masih perlu Anda tingkatkan?
\end{latihan}

\begin{asesmen}
Asesmen Bab 13 (Praktikum 7) merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 3.1 dan Sub-CPMK 4.2. Tugas berupa proyek membangun \textbf{Aplikasi Web CRUD} dengan fitur berikut:
\begin{enumerate}
  \item \textbf{Sistem login dan registrasi} (25\%): Menggunakan session dan \code{password\_hash()}/\code{password\_verify()}. Mengukur Sub-CPMK 3.1 dan 4.2.
  \item \textbf{Operasi CRUD} (30\%): Create, Read, Update, Delete pada satu entitas menggunakan MySQLi dengan prepared statement. Mengukur Sub-CPMK 3.1 dan 4.2.
  \item \textbf{Validasi dan sanitasi} (20\%): Validasi input server-side menggunakan \code{filter\_var()}, sanitasi output dengan \code{htmlspecialchars()}, CSRF token pada setiap form. Mengukur Sub-CPMK 4.2.
  \item \textbf{Struktur kode OOP} (15\%): Penggunaan class, properti, metode, visibility yang tepat. Mengukur Sub-CPMK 3.1.
  \item \textbf{File upload} (5\%) (opsional): Validasi tipe dan ukuran. \textbf{Penanganan error} (5\%): try-catch, pesan informatif. Mengukur Sub-CPMK 4.2.
\end{enumerate}

\begin{rangkuman}
Bab ini membahas PHP tingkat menengah sebagai \textbf{Unit Materi Inti} berbasis Sub-CPMK 3.1 dan 4.2: OOP, session/cookie, dan keamanan dasar. Pada bagian OOP, dibahas pengenalan paradigma OOP dan empat pilarnya, class dan object (properti, metode, typed properties), constructor (termasuk constructor promotion PHP 8) dan destructor, inheritance dan method overriding, visibility (public, protected, private) dengan prinsip encapsulation, serta static property dan method.

Pada bagian session dan cookie, dibahas mekanisme session (siklus hidup, penyimpanan, dan keamanan termasuk \code{session\_regenerate\_id()}), perbedaan session vs cookie, serta atribut keamanan cookie (HttpOnly, Secure, SameSite). Pada bagian keamanan, dibahas file upload dengan validasi ketat, penanganan error modern (try-catch, custom exception), koneksi database MySQLi (OOP), query CRUD dengan prepared statement, serta validasi dan sanitasi input (termasuk mitigasi XSS dan CSRF).

\textbf{Poin Kunci:}
\begin{itemize}
  \item OOP mengorganisasi kode PHP ke dalam class dan object; empat pilar: encapsulation, inheritance, polymorphism, abstraction.
  \item Session menyimpan data di server (aman), cookie di client (ringan); keduanya perlu pengaturan keamanan.
  \item Prepared statement mencegah SQL injection; \code{htmlspecialchars()} mencegah XSS; CSRF token mencegah request palsu.
  \item Validasi input dan sanitasi output adalah prinsip fundamental keamanan web (OWASP).
\end{itemize}

\end{rangkuman}
