\begin{aktivitas}
  \item Buat endpoint sederhana yang mengembalikan JSON dan tampilkan di halaman web.
  \item Lakukan pengujian fungsional pada form yang dibuat: skenario positif dan negatif.
  \item Ukur performa halaman menggunakan Lighthouse (DevTools) dan catat skor serta rekomendasi.
  \item Terapkan validasi input di server (PHP) pada form yang sudah dibuat.
  \item Ganti query biasa menjadi prepared statement pada kode PHP yang mengakses database.
  \item Implementasikan CSRF token pada form submit.
  \item Terapkan \code{htmlspecialchars()} pada semua output yang berasal dari input pengguna.
  \item Lakukan audit keamanan sederhana menggunakan checklist OWASP Top 10.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan pengujian fungsional, kompatibilitas, dan performa.
  \item Sebutkan tiga metrik performa web dan alat untuk mengukurnya.
  \item Jelaskan mengapa validasi di client saja tidak cukup untuk keamanan.
  \item Tulis contoh prepared statement dengan MySQLi untuk query INSERT.
  \item Jelaskan perbedaan XSS dan CSRF beserta cara pencegahannya.
  \item Sebutkan tiga atribut cookie untuk keamanan dan fungsi masing-masing.
\end{latihan}

\begin{asesmen}
Asesmen berupa tugas integrasi API, laporan pengujian fungsional dan performa, serta implementasi keamanan (prepared statement, CSRF token, output encoding) pada aplikasi web yang telah dibuat. Dinilai kelengkapan implementasi dan pemahaman konsep keamanan.
\end{asesmen}

\begin{checklist}
  \item Saya dapat melakukan request API menggunakan fetch dan menampilkan hasilnya.
  \item Saya memahami jenis-jenis pengujian: fungsional, kompatibilitas, dan performa.
  \item Saya dapat menggunakan DevTools untuk debugging dan pengukuran performa.
  \item Saya memahami strategi pengujian: manual vs otomatis, unit vs integrasi vs E2E.
  \item Saya mengetahui metrik performa web (FCP, LCP) dan alat pengukurnya.
  \item Saya memahami ancaman keamanan web umum (SQL injection, XSS, CSRF).
  \item Saya dapat menerapkan validasi input dan sanitasi data di server (PHP).
  \item Saya dapat menggunakan prepared statement untuk mencegah SQL injection.
  \item Saya memahami pengelolaan session dan cookie yang aman.
  \item Saya dapat menerapkan pencegahan XSS (output encoding) dan CSRF (token).
  \item Saya memahami konsep load testing dan optimasi performa.
  \item Saya mengenal OWASP Top 10 dan checklist audit keamanan.
  \item Saya memahami session management lanjutan dan mitigasi serangannya.
\end{checklist}

\begin{rangkuman}
Bab ini membahas integrasi API, pengujian aplikasi web secara menyeluruh (fungsional, performa, load testing), serta keamanan web dari dasar hingga lanjut. Keamanan dan pengujian bukan langkah terakhir; keduanya harus terintegrasi di setiap tahap pengembangan.

\textbf{Poin Kunci:}
\begin{itemize}
  \item API memudahkan pertukaran data antara front-end dan back-end.
  \item Pengujian (fungsional, kompatibilitas, performa) memastikan aplikasi andal dan cepat.
  \item Validasi input di server, prepared statement, dan output encoding adalah pertahanan utama terhadap serangan.
  \item Session dan cookie harus dikelola dengan atribut keamanan (HttpOnly, Secure, SameSite).
  \item CSRF token dan regenerasi session ID melindungi dari serangan lintas situs.
  \item Load testing mengidentifikasi bottleneck sebelum deployment; optimasi (caching, minifikasi, lazy loading) mempercepat aplikasi.
  \item Audit keamanan berbasis OWASP Top 10 menjaga aplikasi tetap aman di produksi.
\end{itemize}
\end{rangkuman}
