\section{Pola Desain Dasar: Module Pattern dan Singleton}

\textbf{Module pattern} membungkus kode dalam IIFE (Immediately Invoked Function Expression) atau closure untuk menciptakan scope privat. Variabel dan fungsi di dalam module tidak mencemari scope global. Hanya API publik yang di-return. Dengan ES6 modules (\code{import}/\code{export}), module pattern native sudah tersedia, tetapi IIFE pattern masih relevan untuk pemahaman konsep \cite{eloquentJs}.

\textbf{Singleton pattern} memastikan hanya ada satu instance dari suatu objek di seluruh aplikasi---berguna untuk konfigurasi, state global, atau koneksi database. Dalam JavaScript, singleton sering di-implementasi sebagai objek literal atau modul yang mengexport satu instance \cite{jsInfo}.

\begin{catatan}
\textbf{Pola Desain Bukan Harga Mati.} Pola desain adalah solusi terbukti untuk masalah umum, bukan aturan wajib. Gunakan pola ketika masalah sesuai; jangan memaksakan pola demi pola. Untuk proyek JavaScript modern, ES6 modules sudah menyelesaikan banyak masalah yang dulu memerlukan module pattern manual. Observer pattern diimplementasikan oleh \code{addEventListener}; Iterator pattern oleh \code{for...of} \cite{mdnJsGuide}.
\end{catatan}
