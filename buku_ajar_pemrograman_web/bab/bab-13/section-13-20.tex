\section{Session Management Lanjutan: Fixation, Hijacking, dan Mitigasi}

\textbf{Session fixation} terjadi saat penyerang memaksa korban menggunakan session ID yang sudah diketahui penyerang --- setelah korban login, penyerang menggunakan session ID tersebut untuk mengakses akun korban. Mitigasi: selalu panggil \code{session\_regenerate\_id(true)} setelah login berhasil. \textbf{Session hijacking} terjadi saat penyerang mencuri session ID aktif (melalui XSS, penyadapan jaringan, atau malware). Mitigasi: (1) cookie HttpOnly (cegah XSS); (2) cookie Secure (cegah penyadapan); (3) SameSite (cegah CSRF); (4) timeout session \cite{phpRightWay}.

\begin{webcode}[language=PHP, caption={Mitigasi session fixation dan hijacking}]
<?php
// Konfigurasi session SEBELUM session_start()
ini_set('session.cookie_httponly', 1);  // Cegah XSS
ini_set('session.cookie_secure', 1);    // Hanya HTTPS
ini_set('session.cookie_samesite', 'Strict');
ini_set('session.use_strict_mode', 1);  // Tolak ID buatan
ini_set('session.gc_maxlifetime', 1800); // Timeout 30 menit
session_start();

// Setelah login: regenerasi ID (cegah fixation)
session_regenerate_id(true);

// Cek timeout
if (isset($_SESSION['last_activity'])
    && (time() - $_SESSION['last_activity']) > 1800) {
  session_unset();
  session_destroy();
  header("Location: login.php?timeout=1");
  exit;
}
$_SESSION['last_activity'] = time();
?>
\end{webcode}
