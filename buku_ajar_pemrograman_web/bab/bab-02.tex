\chapter{Pengantar Pemrograman Web, Arsitektur Client-Server, Ruang Lingkup Kebutuhan Pengguna}

Bab ini menyajikan tiga materi pokok sesuai \textbf{Pertemuan 1} RPS: (1) \textbf{pengantar pemrograman web}, pengenalan konteks dan teknologi web; (2) \textbf{arsitektur client-server}, fondasi komunikasi antara browser dan server; serta (3) \textbf{ruang lingkup kebutuhan pengguna}, identifikasi kebutuhan fungsional dan non-fungsional untuk solusi web (Sub-CPMK 1.1). Pemahaman ketiga materi ini menjadi prasyarat sebelum masuk ke analisis kebutuhan lanjutan (Pertemuan 2) dan implementasi HTML/CSS (Pertemuan 4--7).

\begin{aktivitas}
  \item \textbf{Studi kasus:} Identifikasi kebutuhan fungsional dan non-fungsional untuk sebuah aplikasi web pilihan (misalnya toko online sederhana). Buat daftar minimal 5 kebutuhan fungsional dan 3 kebutuhan non-fungsional, lalu jelaskan perbedaannya.
  \item \textbf{Praktik:} Buka browser, ketik sebuah URL (misalnya \url{https://www.example.com/halaman?id=1#bagian}), lalu amati request dan response di tab Network pada DevTools. Identifikasi metode HTTP, status code, header, dan komponen URL yang terlihat.
  \item \textbf{Praktik:} Instal XAMPP di komputer Anda. Buat file \code{info.php} berisi \code{<?php phpinfo(); ?>} di folder \code{htdocs}, lalu akses melalui \code{http://localhost/info.php}. Catat versi PHP dan modul yang aktif.
  \item \textbf{Praktik:} Buat halaman HTML sederhana yang menampilkan judul, paragraf, dan tombol. Tambahkan CSS untuk mengubah warna, font, dan tata letak menggunakan Flexbox. Tambahkan JavaScript agar tombol mengubah teks paragraf saat diklik.
  \item \textbf{Praktik:} Buat file PHP yang menghubungkan ke database MySQL menggunakan \code{mysqli}. Buat tabel \code{produk} di phpMyAdmin, isi dengan beberapa data, lalu tampilkan data tersebut di halaman web.
  \item \textbf{Latihan:} Gambar peta konsep hubungan antar teknologi web (HTML, CSS, JS, PHP, MySQL, HTTP, DNS, API) di kertas atau alat digital. Tunjukkan arah aliran data dan identifikasi mana yang berjalan di client dan mana di server.
  \item \textbf{Praktik:} Lakukan audit aksesibilitas dan performa menggunakan Lighthouse di Chrome DevTools pada sebuah situs web pilihan Anda. Catat skor dan tiga rekomendasi perbaikan yang paling penting.
  \item \textbf{Praktik:} Inisialisasi repositori Git untuk folder proyek latihan. Buat commit pertama, lalu buat branch baru untuk menambahkan fitur. Praktikkan perintah \code{git add}, \code{git commit}, \code{git branch}, dan \code{git merge}.
  \item \textbf{Praktik:} Buat contoh kode PHP yang rentan terhadap SQL Injection, lalu perbaiki menggunakan prepared statement. Uji kedua versi untuk memahami perbedaannya.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan kebutuhan fungsional dan non-fungsional. Berikan masing-masing tiga contoh untuk aplikasi web toko online (Sub-CPMK 1.1).
  \item Jelaskan perbedaan peran client dan server dalam arsitektur web. Berikan contoh konkret masing-masing dan gambarkan alur komunikasinya.
  \item Sebutkan dan jelaskan tujuh komponen URL beserta contohnya. Berikan contoh URL lengkap dan identifikasi setiap komponennya.
  \item Jelaskan perbedaan metode HTTP GET dan POST. Kapan masing-masing digunakan? Apa yang dimaksud dengan \textit{idempotent}?
  \item Sebutkan lima status code HTTP dari kategori yang berbeda (2xx, 3xx, 4xx, 5xx) beserta artinya. Berikan situasi nyata di mana masing-masing muncul.
  \item Jelaskan peran HTML, CSS, dan JavaScript dalam pembangunan halaman web. Apa yang dimaksud dengan \textit{separation of concerns} dan mengapa prinsip ini penting?
  \item Apa perbedaan antara front-end dan back-end? Jelaskan alur lengkap dari saat pengguna mengklik tombol hingga data ditampilkan di halaman.
  \item Sebutkan dan jelaskan secara singkat enam tahapan siklus pengembangan web (SDLC). Apa output utama dari masing-masing tahap?
  \item Jelaskan perbedaan pendekatan Waterfall dan Agile dalam pengembangan web. Kapan masing-masing lebih tepat digunakan?
  \item Jelaskan tiga ancaman keamanan web utama (SQL Injection, XSS, CSRF): bagaimana serangannya bekerja dan cara pencegahannya.
  \item Apa yang dimaksud dengan aksesibilitas web? Sebutkan empat prinsip WCAG (POUR) dan berikan contoh implementasi masing-masing.
  \item Sebutkan tiga metrik Core Web Vitals dan jelaskan apa yang diukur oleh masing-masing. Apa target yang dianggap baik?
  \item Sebutkan tiga organisasi standar web dan kontribusi masing-masing. Mengapa standar web penting untuk interoperabilitas?
\end{latihan}

\begin{asesmen}
Asesmen bab ini berupa:
\begin{enumerate}
  \item \textbf{Kuis konsep}: pertanyaan pilihan ganda dan isian singkat tentang kebutuhan fungsional dan non-fungsional (Sub-CPMK 1.1), arsitektur web, HTTP (metode, status code, header), teknologi front-end/back-end, SDLC, standar web, dan keamanan dasar.
  \item \textbf{Tugas praktik}: buat halaman web (HTML + CSS + JS) yang menampilkan daftar produk dengan tata letak Flexbox. Halaman harus berjalan di server lokal XAMPP. Sertakan screenshot DevTools yang menunjukkan:
  \begin{itemize}
    \item Request HTTP dan status code di tab Network
    \item Struktur HTML di tab Elements
    \item Hasil audit Lighthouse (skor Accessibility dan Performance)
  \end{itemize}
  \item \textbf{Tugas back-end}: buat file PHP yang mengambil data produk dari database MySQL dan menampilkannya di halaman. Gunakan prepared statement untuk query database.
  \item \textbf{Peta konsep}: gambar diagram hubungan antar teknologi web yang dipelajari di bab ini, tunjukkan alur data dari browser ke database dan kembali.
\end{enumerate}
\end{asesmen}

\begin{checklist}
  \item Saya dapat mengidentifikasi kebutuhan fungsional dan non-fungsional pengguna untuk solusi web (Sub-CPMK 1.1).
  \item Saya memahami arsitektur client-server, peran DNS, dan siklus request-response HTTP.
  \item Saya dapat menjelaskan komponen URL (skema, host, port, path, query, fragment).
  \item Saya dapat menjelaskan metode HTTP (GET, POST, PUT, DELETE, PATCH) dan konsep idempotent.
  \item Saya dapat membaca dan menginterpretasikan status code HTTP (2xx, 3xx, 4xx, 5xx).
  \item Saya memahami peran HTTP header dan perbedaan HTTP vs HTTPS (TLS/SSL).
  \item Saya dapat menjelaskan peran HTML, CSS, dan JavaScript, termasuk konsep Box Model dan Flexbox.
  \item Saya memahami peran PHP, MySQL, dan REST API di sisi back-end beserta alur pemrosesannya.
  \item Saya mengenal tahapan siklus pengembangan web (SDLC) dan perbedaan Waterfall vs Agile.
  \item Saya dapat menggunakan alat pengembangan: VS Code, DevTools, XAMPP, dan Git.
  \item Saya mengenal organisasi standar web (W3C, WHATWG, ECMA, IETF) dan perannya.
  \item Saya memahami hubungan antar teknologi web melalui peta konsep, termasuk konsep AJAX/Fetch.
  \item Saya mengenal tiga ancaman keamanan utama (SQL Injection, XSS, CSRF) dan teknik pencegahannya.
  \item Saya memahami prinsip defense in depth dan pentingnya validasi server-side.
  \item Saya memahami prinsip aksesibilitas web (WCAG, POUR) dan penggunaan HTML semantik serta ARIA.
  \item Saya mengenal metrik Core Web Vitals (LCP, INP, CLS) dan cara mengukurnya dengan Lighthouse.
\end{checklist}

\begin{rangkuman}
Bab ini menyajikan landasan teori dan konsep dasar yang menjadi fondasi seluruh pembelajaran pemrograman web dalam buku ini.

\textbf{Poin Kunci:}
\begin{itemize}
  \item \textbf{Ruang lingkup kebutuhan pengguna} mencakup kebutuhan fungsional (apa yang dilakukan sistem) dan non-fungsional (bagaimana sistem berperilaku). Identifikasi kebutuhan adalah fondasi pengembangan solusi web (Sub-CPMK 1.1).
  \item \textbf{Arsitektur web} didasarkan pada model client-server dengan HTTP sebagai protokol komunikasi. DNS menerjemahkan nama domain menjadi alamat IP. URL mengidentifikasi sumber daya melalui komponen: skema, host, port, path, query, dan fragment. Koneksi dimulai dengan TCP three-way handshake.
  \item \textbf{HTTP} memiliki metode (GET, POST, PUT, DELETE, PATCH), status code (2xx, 3xx, 4xx, 5xx), dan header yang membawa metadata. HTTP bersifat stateless; cookie dan session digunakan untuk mengelola state. HTTPS menambahkan enkripsi TLS untuk keamanan.
  \item \textbf{Front-end} terdiri dari HTML (struktur), CSS (presentasi), dan JavaScript (interaktivitas). CSS modern menyediakan Flexbox dan Grid untuk layout responsif. JavaScript ES6+ menambahkan fitur seperti arrow function, Promise, dan Fetch API.
  \item \textbf{Back-end} terdiri dari PHP (logika bisnis), MySQL (penyimpanan data), dan web server (Apache). REST API menggunakan format JSON untuk komunikasi front-end dan back-end.
  \item \textbf{Siklus pengembangan web} meliputi analisis kebutuhan, perancangan, implementasi, pengujian, deployment, dan pemeliharaan. Pendekatan Agile memungkinkan iterasi cepat melalui sprint.
  \item \textbf{Alat pengembangan} yang diperlukan meliputi editor kode (VS Code), browser DevTools, server lokal (XAMPP), version control (Git), dan terminal.
  \item \textbf{Standar web} dikembangkan oleh W3C (CSS), WHATWG (HTML), ECMA (JavaScript), dan IETF (HTTP). Standar memastikan interoperabilitas lintas browser dan platform.
  \item \textbf{Peta konsep} menggambarkan hubungan antar teknologi web. AJAX/Fetch memungkinkan JavaScript berkomunikasi dengan server tanpa reload halaman.
  \item \textbf{Keamanan web} harus diperhatikan sejak awal: waspadai SQL Injection (gunakan prepared statement), XSS (gunakan output encoding), dan CSRF (gunakan CSRF token). Prinsip utama: \textit{never trust user input}.
  \item \textbf{Aksesibilitas} (WCAG/POUR) dan \textbf{performa} (Core Web Vitals: LCP, INP, CLS) merupakan aspek kualitas yang meningkatkan jangkauan pengguna dan pengalaman browsing.
\end{itemize}
\end{rangkuman}
