\begin{subcpmk}
  \item Sub-CPMK 1.1: Mengidentifikasi kebutuhan fungsional dan non-fungsional pengguna untuk solusi web.
  \item Sub-CPMK 1.2: Membuat wireframe dan mockup antarmuka web yang mendukung alur pengguna.
  \item Sub-CPMK 1.3: Menerapkan prinsip aksesibilitas (a11y) dan pengalaman pengguna (UX).
\end{subcpmk}

\begin{aktivitas}
  \item Susun daftar kebutuhan fungsional dan non-fungsional dari studi kasus sederhana.
  \item Buat wireframe dua halaman utama menggunakan kertas atau alat desain.
  \item Evaluasi wireframe dengan prinsip aksesibilitas dasar.
\end{aktivitas}

\begin{latihan}
  \item Buat tiga kebutuhan fungsional dan tiga kebutuhan non-fungsional untuk aplikasi catatan harian.
  \item Buat user flow untuk fitur ``Tambah Catatan''.
\end{latihan}

\begin{asesmen}
Asesmen berupa penilaian wireframe dan daftar kebutuhan yang disusun mahasiswa.
\end{asesmen}

\begin{checklist}
  \item Saya dapat membedakan kebutuhan fungsional dan non-fungsional.
  \item Saya dapat membuat wireframe dan user flow sederhana.
  \item Saya memahami prinsip aksesibilitas dasar.
\end{checklist}

\begin{rangkuman}
Bab ini menekankan pentingnya analisis kebutuhan serta perancangan antarmuka sebelum coding. Dengan wireframe, mockup, dan aksesibilitas, Anda memastikan solusi yang dibangun sesuai kebutuhan pengguna.

\textbf{Poin Kunci:}
\begin{itemize}
  \item Kebutuhan fungsional dan non-fungsional harus didefinisikan sejak awal.
  \item Wireframe dan mockup membantu memvalidasi desain sebelum implementasi.
  \item Aksesibilitas membuat aplikasi lebih inklusif dan mudah digunakan.
\end{itemize}
\end{rangkuman}
