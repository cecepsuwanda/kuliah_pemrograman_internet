\chapter{Analisis Kebutuhan, User Flow, dan Wireframe Dasar}

Bab ini membahas analisis kebutuhan (fungsional dan non-fungsional), pemahaman pengguna melalui persona dan skenario, pengenalan UI/UX, serta perancangan wireframe dan user flow. Materi ini mendukung pencapaian Sub-CPMK 1.1 dan 1.2 sesuai rencana pembelajaran pertemuan kedua. Pengembangan ke mockup, prototyping, aksesibilitas, dan prinsip UX lanjutan dibahas di Bab 4.

\begin{subcpmk}
  \item Sub-CPMK 1.1: Mengidentifikasi kebutuhan fungsional dan non-fungsional pengguna untuk solusi web.
  \item Sub-CPMK 1.2: Membuat wireframe dan mockup antarmuka web yang mendukung alur pengguna (user flow).
\end{subcpmk}

\begin{aktivitas}
  \item \textbf{Studi kasus:} Pilih sebuah aplikasi web (mis. toko online, reservasi perpustakaan). Identifikasi tiga pemangku kepentingan (\textit{stakeholder}) dan lakukan wawancara singkat untuk menggali kebutuhan.
  \item \textbf{Praktik:} Susun daftar kebutuhan fungsional dan non-fungsional dari hasil wawancara, lalu prioritaskan menggunakan metode MoSCoW.
  \item \textbf{Latihan:} Buat dua persona pengguna lengkap (dengan tujuan, frustrasi, dan konteks penggunaan) berdasarkan studi kasus.
  \item \textbf{Praktik:} Buat wireframe low-fidelity (kertas atau digital) untuk tiga halaman utama: beranda, form input, dan halaman detail.
  \item \textbf{Praktik:} Gambar user flow lengkap untuk dua fitur utama, termasuk percabangan error handling.
  \item \textbf{Diskusi:} Presentasikan wireframe dan user flow di depan kelompok; terima umpan balik dan lakukan iterasi.
\end{aktivitas}

\begin{latihan}
  \item Jelaskan perbedaan antara kebutuhan fungsional dan non-fungsional. Berikan masing-masing tiga contoh untuk aplikasi toko online.
  \item Buat tiga \textit{acceptance criteria} dalam format Given--When--Then untuk fitur ``Tambah Produk ke Keranjang''.
  \item Apa perbedaan UI dan UX? Berikan analogi dari kehidupan sehari-hari.
  \item Buat persona pengguna untuk aplikasi reservasi perpustakaan.
  \item Buat user flow untuk fitur ``Registrasi Akun Baru'', termasuk validasi email dan konfirmasi password.
  \item \textbf{Refleksi:} Dari materi bab ini, manakah indikator Sub-CPMK 1.1 dan 1.2 yang sudah Anda kuasai? Area mana yang masih perlu Anda tingkatkan?
\end{latihan}

\begin{asesmen}
Asesmen Bab 3 (Tugas 1) merupakan \textbf{Evaluasi Kinerja} yang mengukur Sub-CPMK 1.1 dan Sub-CPMK 1.2. Tugas berupa dokumen dan artefak yang mencakup:
\begin{enumerate}
  \item \textbf{Dokumen Kebutuhan} (40\%): Daftar kebutuhan fungsional (min. 5) dan non-fungsional (min. 3) dengan prioritisasi MoSCoW dan acceptance criteria. Mengukur Sub-CPMK 1.1.
  \item \textbf{Persona dan User Flow} (30\%): Dua persona lengkap dan user flow untuk dua fitur utama. Mengukur Sub-CPMK 1.1 dan 1.2.
  \item \textbf{Wireframe} (30\%): Wireframe low-fidelity untuk tiga halaman utama (beranda, form input, halaman detail). Mengukur Sub-CPMK 1.2.
\end{enumerate}

\textbf{Rubrik Penilaian (berbasis OBE):}
\begin{itemize}
  \item \textbf{A (85--100)}: Seluruh komponen lengkap, kebutuhan terdokumentasi dengan jelas, wireframe dan user flow profesional. Kompeten untuk Sub-CPMK 1.1 dan 1.2.
  \item \textbf{B (70--84)}: Sebagian besar komponen lengkap, beberapa detail kurang.
  \item \textbf{C (55--69)}: Komponen dasar ada tetapi kurang mendalam.
  \item \textbf{D ($<$55)}: Komponen tidak lengkap atau tidak memenuhi standar minimum.
\end{itemize}
\end{asesmen}

\begin{rangkuman}
Bab ini membahas fondasi analisis kebutuhan dan perancangan awal antarmuka sebagai Unit Materi Inti berbasis Sub-CPMK 1.1 dan 1.2: penggalian kebutuhan, kebutuhan fungsional dan non-fungsional, persona dan skenario, pengenalan UI/UX, wireframe, dan user flow.

\textbf{Poin Kunci:}
\begin{itemize}
  \item \textbf{Analisis kebutuhan} (Sub-CPMK 1.1): identifikasi stakeholder, lakukan elicitation (wawancara, observasi, survei), dokumentasikan kebutuhan fungsional dan non-fungsional dengan prioritisasi MoSCoW dan acceptance criteria.
  \item \textbf{Persona dan skenario} (Sub-CPMK 1.1): mewakili pengguna nyata; keputusan desain harus didasarkan pada kebutuhan mereka, bukan asumsi tim.
  \item \textbf{UI} (tampilan) dan \textbf{UX} (pengalaman) saling melengkapi; produk terbaik unggul di keduanya.
  \item \textbf{Wireframe} (Sub-CPMK 1.2): low-fi memvalidasi struktur dan tata letak sebelum detail visual; lanjutkan ke mockup dan prototipe di Bab 4.
  \item \textbf{User flow} (Sub-CPMK 1.2): diagram alur interaksi dari entry point hingga goal, termasuk percabangan dan penanganan error.
\end{itemize}

\textbf{Refleksi:} Pastikan Anda dapat mencentang checklist di atas dan siap melanjutkan ke Bab 4 (mockup, prototyping, aksesibilitas, prinsip UX).
\end{rangkuman}
